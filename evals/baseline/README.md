# Baseline eval-корзина

Каталог фиксирует сценарии, конфигурацию и формат отчёта до дальнейших изменений
harness.

- `scenarios.jsonl` — декларативные eval fixtures.
- `harness_snapshot.json` — версия кода, prompts, конфигурации и состава tools.
- `BASELINE_REPORT.md` — таблица метрик фактических прогонов.
- `KNOWN_FAILURE_MODES.md` — известные риски исходной реализации.

Фактический baseline нельзя считать снятым, пока каждый сценарий не выполнен
одной и той же моделью и результаты не занесены в `BASELINE_REPORT.md`.
Автоматический запуск не должен читать ключи из `model.py` или переменных окружения
без отдельного явного действия пользователя.
