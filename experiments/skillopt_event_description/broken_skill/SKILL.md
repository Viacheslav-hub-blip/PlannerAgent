# Поиск записей по смысловому описанию

Используй этот skill, когда пользователь просит найти, посчитать, проверить или проанализировать срабатывания/операции по смыслу текстового описания.

Базовая таблица для срабатываний обычно `hits`. Основное поле для поиска по смыслу - `event_description`.

## Алгоритм

1. Определи период из запроса пользователя. Если период не указан, используй весь доступный период только когда он явно требуется пользователю.
2. Для количественных вопросов всегда загружай данные через `data-retrieval-agent` или `load_data`; не отвечай по смыслу без фактов.
3. Сначала получи строки или значения `event_description` за нужный период из таблицы `hits`.
4. Выбери точные значения `event_description`, которые соответствуют смысловой категории пользователя.
5. Для финального расчета фильтруй данные по выбранным точным значениям `event_description`, а не по приблизительным ключевым словам.
6. Верни количество, сумму или другие метрики только после проверки результата на загруженных данных.

## Пример первичной загрузки

```text
LOAD hits
PERIOD event_dt FROM '20260101' TO '20260131'
SELECT event_description, event_dt
```

## Ограничения

- Не делай выводы по preview, если нужен точный count или сумма.
- Не заменяй найденные точные значения `event_description` на `LIKE`, `CONTAINS` или похожие подстроки в финальном расчете.
- Если пользователь спрашивает про образовательные оплаты, учитывай близкие формулировки: обучение, онлайн-курсы, образовательная платформа, экзамен, репетитор, учебные материалы.
