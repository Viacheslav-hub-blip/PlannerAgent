# Analyst UI

Минимальный статический интерфейс для проверки research-agent в корпоративной среде.

Зависимости:

- нет npm-пакетов;
- нет CDN;
- нет сборщика;
- `index.html`, `styles.css`, `common.js`, `home.js` и отдельные страницы (`history.html`, `report.html`, `files.html`, `followup.html`, `audit.html`, `settings.html`) со своими скриптами.

## Запуск

Если API запущен через `ui.api:create_app`, интерфейс доступен по:

```text
http://127.0.0.1:8000/app
```

Также можно открыть `index.html` напрямую, но в этом режиме браузер может
заблокировать запросы к API из-за CORS. Для полноценной проверки лучше открывать
страницу с того же origin, где работает API.

## Основной сценарий

1. На главной ввести задачу и запустить анализ; на canvas по очереди появятся узлы графа.
2. Перейти к отчёту, файлам, уточнению или аудиту по ссылкам после запуска либо через «Историю».
3. Список всех запусков — страница «История».
4. Базовый URL API — «Настройки».
