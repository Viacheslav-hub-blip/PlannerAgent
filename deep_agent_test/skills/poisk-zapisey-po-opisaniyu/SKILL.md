# Поиск записей по смысловому описанию

## Что делает skill

Skill помогает проанализировать записи по смысловой категории (текстовым полям) из запроса пользователя.

Примеры запросов:

```text
Найди сработки, связанные с оплатой доставки за январь 2025 года.
Посчитай операции, похожие на оплату такси.
Выгрузи записи, относящиеся к подпискам.
Найди события по оплате маркетплейсов.
Сколько было операций, связанных с доставкой еды?
и другие  запросы связанные с анализом не структурированных данных
```

## Как действовать

### Шаг 1. Определи источник и доступные поля

Выбери таблицу или источник, где есть данные для поиска.

Нужно проверить наличие полей
Используй только поля, подтвержденные в schema, preloaded context, skill-описании или tool output.

### Шаг 2. Построй справочник описаний за период

Сначала получи справочник уникальных значений нужного поля за нужный период:

Цель шага — увидеть реальные варианты описаний и понять, какие структурированные признаки с ними связаны.

### Шаг 3. Выбери релевантные признаки по смыслу

На основе загруженных в контекст llm  данных выбери кандидатов, которые относятся к категории пользователя.

### Шаг 4. Получи полную выгрузку
после определения того какие значения данных подходят сделай выгрузку данных только с этими значениями из первоначальной таблицы за нужный период

### Шаг 5. Посчитай результат

Передай полную выгрузку в python-analysis-agent и посчиатй количество

## Зачем нужен такой подход

Такой workflow нужен, чтобы агент находил записи по смыслу, а не по случайному совпадению (вхлждению) слов.

Например `event_description` используется для понимания категории и выбора признаков. Основной отбор строится по проверяемым структурированным полям: кодам операций, MCC и подтвержденным merchant-признакам.


## Ожидаемый финальный ответ

```text
Вывод:
Найдено <N> записей, связанных с <category>, за период <period>.

Что искали:
<смысловая категория пользователя>.

Как отбирали:
Сначала был построен справочник уникальных event_description за период с кодами операций и доступными структурированными признаками.
Затем были выбраны признаки, которые по смыслу соответствуют категории пользователя.
После этого была выполнена полная выгрузка по выбранным структурированным признакам.

Использованные источники:
- Таблица: <table_name>
- Период: <period>
- Поля: <fields>

Критерии отбора:
- Коды операций или событий: <codes>
- MCC: <mcc_values, если поле доступно>
- Merchant-признаки: <top_merchants, если доступны>

Расчеты:
- Всего записей: <N>
- По кодам операций: <...>
- По MCC: <... если поле доступно>
- По мерчантам: <... если поле доступно>
```
