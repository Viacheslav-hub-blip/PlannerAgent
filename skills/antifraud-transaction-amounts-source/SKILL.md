# Источник сумм транзакций

## `source_3.csv`

**Роль:** источник сумм транзакций клиента.

**Абсолютный путь:** не указан в исходном описании. Используй путь из доступного окружения выполнения.

**Бизнес-смысл:** содержит суммы транзакций по клиенту. Используется для оценки финансового эффекта, риск-аналитики и анализа денежного масштаба операций.

# Поля

| Поле | Тип | Значение |
|---|---|---|
| `epk_id` | `string` | Идентификатор клиента. Используется для соединения с другими источниками. |
| `transaction_amount` | `numeric` | Сумма транзакции. |

# Правила интерпретации

- `transaction_amount` используется для расчета суммарного объема операций.
- Источник можно использовать для оценки финансового эффекта по клиенту, сегменту, правилу или CSI-группе.
- При наличии нескольких строк на одного клиента необходимо явно определить агрегацию: сумма, среднее, максимум, медиана или количество операций.
- Не предполагай, что одна строка соответствует одному клиенту, пока это не проверено.

# Типовые вопросы

Используй источник, чтобы ответить:

- какой объем операций связан с клиентом;
- какой финансовый эффект у сегмента;
- какие суммы связаны с конкретным правилом;
- отличаются ли суммы у клиентов с негативным и положительным CSI;
- какие клиенты или группы дают наибольший денежный объем.
