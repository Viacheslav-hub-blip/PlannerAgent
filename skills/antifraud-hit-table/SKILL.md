# Таблица антифрод-сработок

## `cspfs_repo_features3.hits_extra_info_129372427_view`

**Роль:** основная таблица антифрод-сработок.

**Бизнес-смысл:** таблица содержит не все клиентские операции, а только операции, по которым сработал фрод-мониторинг. Это обогащенный слой с информацией о сработке, решении антифрода, правилах, резолюциях и признаках сохраненного ущерба.

Используй таблицу как основной источник для анализа:

- конкретной сработки по `event_id`;
- причины реакции антифрода;
- сработавшего правила;
- решения антифрод-системы;
- резолюции по операции;
- признака предотвращенного ущерба;
- связи с клиентскими событиями до и после сработки.

# Типовые поля

| Поле | Значение | Как использовать |
|---|---|---|
| `event_id` | Идентификатор события / операции | Основной ключ для поиска конкретной сработки. |
| `event_dt` | Дата операции | Используется для фильтрации и восстановления таймлайна. |
| `event_time` | Время операции | Используется для точного порядка событий. |
| `event_channel` | Канал события | Помогает выбрать сырую таблицу событий: `cards_event` или `uko_event`. |
| `sub_channel` | Подканал | Уточняет источник и сценарий операции. |
| `event_type` | Тип события | Помогает понять вид операции. |
| `sub_type` | Подтип события | Используется для более детальной классификации. |
| `type_operation` | Тип операции | Особенно полезно для UKO / не карточных событий. |
| `transaction_amount_in_rub` | Сумма операции в рублях | Используется для денежного анализа. |
| `client_balance` | Баланс клиента | Может использоваться при оценке потенциального ущерба. |
| `purpose` | Назначение операции | Используется для бизнес-группировки операций. |
| `surface` | Поверхность / интерфейс операции | Помогает определить канал и сценарий. |
| `product` | Продукт | Используется для продуктового разреза. |
| `payment_transaction_flag` | Признак платежной операции | Помогает отделить платежные операции от неплатежных. |
| `policy_action` | Решение антифрод-системы | Например: `allow`, `review`, `deny`, `NULL`. |
| `main_rule` | Информация о сработавшем правиле | Используется для объяснения причины сработки. |
| `resolution_first` | Первая резолюция | Используется для анализа первичного результата подтверждения. |
| `resolution_last` | Последняя резолюция | Используется как финальный доступный результат. |
| `accept_time_sec` | Время подтверждения | Используется для оценки клиентского трения. |
| `is_save` | Признак предотвращенного ущерба | Основной флаг сохраненного ущерба. |
| `marked_as_not_save_reason` | Причина исключения из save | Объясняет, почему сработка не считается предотвращенным ущербом. |
| `has_claim` | Признак клиентской жалобы | Используется для связи с клиентским негативом. |
| `previous_events` | События до сработки | Краткий производный контекст, не замена полной истории. |
| `posterious_events` | Успешные исходящие операции после сработки | Краткий производный контекст после сработки. |

# Важные правила интерпретации

- `event_id` — основной идентификатор сработки.
- `policy_action` — решение антифрод-системы, а не финальная резолюция.
- `resolution_first` и `resolution_last` — результаты подтверждения или расследования.
- `is_save` — основной готовый признак предотвращенного ущерба.
- `previous_events` и `posterious_events` — компактный контекст, а не полная история клиента.
- Не считай таблицу сработок полной историей операций клиента.
- Для проверки поведения клиента до и после сработки переходи к raw event tables: `cards_event` или `uko_event`.
