# Источник сегментации и правил

## `source_2.csv`

**Роль:** справочник клиентских сегментов и антифрод-правил.

**Абсолютный путь:** не указан в исходном описании. Используй путь из доступного окружения выполнения.

**Бизнес-смысл:** содержит сегмент клиента и название правила. Может использоваться как базовый справочник для разрезов аналитики и интерпретации антифрод-правил.

# Поля

| Поле | Тип | Значение |
|---|---|---|
| `epk_id` | `string` | Идентификатор клиента. Используется для соединения с другими источниками. |
| `segment` | `string` | Сегмент клиента. Возможные значения: `молодеж`, `пожилые`, `активные`. |
| `rule_name` | `string` | Название правила. Возможные значения: `правило 1`, `правило 2`. |

# Правила интерпретации

- `segment` используется для анализа клиентских групп.
- `rule_name` используется для разреза по антифрод-правилам.
- Источник можно использовать для проверки, какие правила чаще связаны с негативным CSI, суммами транзакций или сработками.

# Типовые вопросы

Используй источник, чтобы ответить:

- к какому сегменту относится клиент;
- какое правило связано с клиентом;
- отличаются ли показатели по сегментам;
- какие правила чаще встречаются у клиентов с негативным CSI;
- какие правила связаны с наибольшими суммами операций.
