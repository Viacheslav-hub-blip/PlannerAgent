# Таблица не карточных событий

## `csp_afpc_sss_inc.uko_event`

**Роль:** таблица полной истории клиентских событий в не карточных каналах.

**Бизнес-смысл:** таблица содержит клиентские события вне специализированного карточного канала.

Используй таблицу, если операция относится к не карточному каналу:

- ДБО;
- ВСП;
- СБОЛ;
- СБП;
- переводы;
- сервисные операции;
- операции в отделении;
- операции со счетами и вкладами;
- изменения профиля;
- подтверждения операций;
- другие не карточные клиентские события.

# Для чего используется

- поиск исходного не карточного события по сработке;
- восстановление поведения клиента до и после сработки;
- проверка повторных переводов после блокировки;
- анализ получателя, реквизитов и типа перевода;
- анализ контекста подтверждения операции;
- проверка, что не карточная сработка представлена в сырой таблице событий.

# Признаки не карточного канала

Используй `uko_event`, если:

- `event_channel` указывает на ДБО, ВСП, СБОЛ, СБП, переводы или сервисные операции;
- `surface` относится к не карточному интерфейсу;
- `type_operation` описывает UKO / не карточную операцию;
- операция является переводом, сервисным действием, изменением профиля, операцией в отделении или подтверждением.

# Ключи для связи

Предпочтительный ключ:

- `event_id`, если он есть в обеих таблицах.

Если `event_id` недоступен или не совпадает, используй составное сопоставление:

- идентификатор клиента;
- дата операции;
- время операции или близкое временное окно;
- сумма операции;
- тип операции;
- реквизиты получателя;
- идентификатор операции, если доступен.

# Важные правила

- Не используй `uko_event` для карточных событий, если они хранятся в `cards_event`.
- Не считай каждую строку `uko_event` антифрод-сработкой.
- Событие в `uko_event` может отсутствовать в таблице сработок, если антифрод не сработал.
- Таблица нужна для анализа полного не карточного контекста клиента.
