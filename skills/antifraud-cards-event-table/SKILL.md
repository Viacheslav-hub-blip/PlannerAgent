# Таблица карточных событий

## `csp_afpc_sss_inc.cards_event`

**Роль:** таблица полной истории клиентских событий в карточном канале.

**Бизнес-смысл:** таблица содержит все карточные события клиента, а не только антифрод-сработки.

Используй таблицу, если операция относится к карточному каналу:

- карточная покупка;
- POS-операция;
- E-commerce-операция;
- ATM-операция;
- карточная авторизация;
- проверка карты;
- токенизация;
- операция с NFC-токеном;
- другой карточный сценарий.

# Для чего используется

- поиск исходного карточного события по сработке;
- восстановление поведения клиента до и после сработки;
- проверка повторных карточных операций;
- сравнение суммы сработки с другими карточными операциями;
- анализ merchant / TST / ATM / POS-контекста;
- проверка, что карточная сработка представлена в сырой таблице событий.

# Признаки карточного канала

Используй `cards_event`, если:

- `event_channel` указывает на карты или карточный канал;
- `surface` содержит `POS`, `E-comm`, `ATM` или другую карточную поверхность;
- `product` связан с картой, NFC-токеном или карточным платежным инструментом;
- операция является карточной покупкой, снятием, авторизацией, проверкой карты или токенизацией.

# Ключи для связи

Предпочтительный ключ:

- `event_id`, если он есть в обеих таблицах.

Если `event_id` недоступен или не совпадает, используй составное сопоставление:

- идентификатор клиента;
- дата операции;
- время операции или близкое временное окно;
- сумма операции;
- тип операции;
- карточный идентификатор, если доступен.

# Важные правила

- Не считай каждую строку `cards_event` антифрод-сработкой.
- Не используй `cards_event` для не карточных событий.
- Событие в `cards_event` может отсутствовать в таблице сработок, если антифрод не сработал.
- Таблица нужна для анализа полного карточного контекста клиента.
